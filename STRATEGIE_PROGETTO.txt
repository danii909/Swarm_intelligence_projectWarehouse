────────────────────────────────────────────────────────────────────────────────
1. STRATEGIE DI ESPLORAZIONE (5 strategie)
────────────────────────────────────────────────────────────────────────────────

Tutte ereditano da ExplorationStrategy (src/agents/strategies/base.py)
e implementano il metodo next_move().

┌─────────────────────────────────────────────────────────────────────────────┐
│ 1. RANDOM WALK — src/agents/strategies/random_walk.py                       │
│                                                                             │
│  - Si sposta in una direzione casuale tra quelle percorribili.              │
│  - Priorità:                                                                │
│      1) Se trasporta oggetto  → si dirige al magazzino più vicino           │
│      2) Se conosce oggetti    → si dirige all'oggetto più vicino (greedy)   │
│      3) Altrimenti            → mossa casuale                               │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ 2. FRONTIER-BASED — src/agents/strategies/frontier.py                       │
│                                                                             │
│  - Mantiene un insieme di "frontiere": celle esplorate adiacenti ad aree    │
│    ancora inesplorate.                                                      │
│  - Si muove sempre verso la frontiera più vicina (distanza Manhattan).      │
│  - Fallback: random walk quando la mappa è completamente esplorata.         │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ 3. SPIRAL / ZIGZAG — src/agents/strategies/spiral.py                        │
│                                                                             │
│  - Percorre la griglia a serpentina riga-per-riga                           │
│    (direzione alternata ogni 2 righe).                                      │
│  - Genera una lista di waypoint precalcolati e li segue in ordine.          │
│  - Se un waypoint è irraggiungibile, lo salta.                              │
│  - Al termine di tutti i waypoint: rally alla cella (0,0).                 │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ 4. SECTOR COVERAGE — src/agents/strategies/sector.py                        │
│                                                                             │
│  - Divide la griglia in settori orizzontali uguali, uno per agente          │
│    (assegnato tramite agent_id % num_agents).                               │
│  - Ogni agente esplora prima il proprio settore.                            │
│  - Quando il settore è completamente esplorato, si espande come             │
│    frontier globale su tutta la griglia.                                    │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ 5. GREEDY — src/agents/strategies/greedy.py                                 │
│                                                                             │
│  - Si dirige direttamente sull'oggetto noto più vicino (puro greedy).       │
│  - Se non conosce oggetti, usa FrontierStrategy come fallback.              │
└─────────────────────────────────────────────────────────────────────────────┘


────────────────────────────────────────────────────────────────────────────────
2. SENSORI DEGLI AGENTI — src/agents/sensors.py
────────────────────────────────────────────────────────────────────────────────

  VISIBILITÀ
  ----------
  - Raggio Manhattan configurabile (default 2, range 1–3 nella config di default).
  - Occlusione simulata con ray-casting Bresenham:
    una cella è visibile solo se nessuna cella intermedia è un muro.

  COMUNICAZIONE (distanza)
  ------------------------
  - Due agenti possono comunicare se la distanza Manhattan tra le
    loro posizioni è ≤ min(comm_radius_A, comm_radius_B) (default 2).


────────────────────────────────────────────────────────────────────────────────
3. PROTOCOLLO DI COMUNICAZIONE — src/communication/protocol.py
────────────────────────────────────────────────────────────────────────────────

  - Comunicazione a ogni tick tra tutte le coppie entro raggio.
  - Merge BIDIREZIONALE delle seguenti informazioni:

      1) local_map      → merge per unione (chi ha più info vince)
      2) known_objects  → unione degli insiemi di oggetti noti
      3) known_agents   → propagazione transitiva delle posizioni;
                          politica "voce più recente vince" per ogni ID

  - Le informazioni sulle posizioni degli agenti scadono dopo 5 tick
    (_MAX_AGENT_INFO_AGE = 5).


────────────────────────────────────────────────────────────────────────────────
4. PATHFINDING — src/pathfinding/pathfinder.py
────────────────────────────────────────────────────────────────────────────────

  Due algoritmi disponibili:

  BFS (Breadth-First Search)
  - Default per griglia 25×25 con ostacoli statici.
  - Garantisce il percorso più breve in numero di passi.

  A* (A-Star)
  - Alternativa con euristica Manhattan per ridurre i nodi espansi.
  - Più efficiente in griglie grandi o con molti ostacoli.

  Entrambi rispettano la direzionalità delle porte:
  - Le celle ENTRANCE/EXIT sono percorribili SOLO come destinazione finale,
    mai come celle di transito intermedio (in modalità normale).
  - In modalità allow_warehouse=True (consegna interna) vengono incluse
    anche le celle WAREHOUSE.


────────────────────────────────────────────────────────────────────────────────
5. STATI INTERNI DEGLI AGENTI — src/agents/agent.py
────────────────────────────────────────────────────────────────────────────────

  EXPLORING ──→ MOVING_TO_OBJECT ──→ CARRYING ──→ DELIVERING
                                                      │
                                              EXITING_WAREHOUSE
                                                      │
                                               (EXPLORING) ← torna qui
                                                      │
                                               DEPLETED (batteria 0)

  Batteria: 500 unità, -1 per ogni spostamento.


────────────────────────────────────────────────────────────────────────────────
6. CICLO DI SIMULAZIONE — src/simulation/simulator.py
────────────────────────────────────────────────────────────────────────────────

  Ad ogni tick il Simulator esegue nell'ordine:

    1. PERCEZIONE        — ogni agente aggiorna la sua local_map
    2. VISIBILITÀ DIR.   — aggiornamento di known_agents per agenti visibili
    3. COMUNICAZIONE     — communicate_agents() tra coppie entro raggio
    4. PIANIFICAZIONE    — ogni agente calcola la prossima mossa
                           (anti-collisione tramite set "occupied")
    5. APPLICA MOSSE     — risoluzione conflitti no-overlap
    6. RACCOLTA/CONSEGNA — pick_up() e deliver()
    7. METRICHE & LOG    — snapshot ogni log_every tick

  Condizioni di terminazione:
    - Tutti gli oggetti consegnati
    - Tutti gli agenti con batteria esaurita (DEPLETED)
    - Raggiunto il numero massimo di tick (max_ticks, default 500)


────────────────────────────────────────────────────────────────────────────────
7. CONFIGURAZIONE DI DEFAULT (5 agenti, uno per strategia)
────────────────────────────────────────────────────────────────────────────────

  Agente 0 → RandomWalk    | visibility_radius = 1
  Agente 1 → Frontier      | visibility_radius = 2
  Agente 2 → Spiral        | visibility_radius = 3
  Agente 3 → Sector        | visibility_radius = 1
  Agente 4 → Greedy        | visibility_radius = 2

  communication_radius = 2 (default per tutti)
